Differences Between Let, Var, Const

Types Scope Reassignment and Redeclaration Hoisting Notable points on each
LET Block-scoped. This means they are only accessible within the specific block (e.g., if statements, for loops, or any code enclosed in curly braces {}) where they are declared. Can be reassigned but cannot be redeclared within the same scope. Hoisted to the top of their block scope but are not initialized. Attempting to access them before their declaration results in a ReferenceError (known as the Temporal Dead Zone or TDZ). Use let for variables that need to be reassigned.
VAR Function-scoped or globally-scoped. This means a var variable is accessible throughout the entire function it's declared in, or globally if declared outside any function. Can be both reassigned (its value can be changed) and redeclared (declared again with the same name within the same scope). Hoisted to the top of its scope and initialized with undefined. This means you can access a var variable before its declaration in the code, and it will have a value of undefined. Avoid using var in modern JavaScript due to its less predictable scoping behavior, which can lead to unexpected bugs.
CONST Block-scoped. This means they are only accessible within the specific block (e.g., if statements, for loops, or any code enclosed in curly braces {}) where they are declared. Cannot be reassigned or redeclared after its initial assignment. For objects and arrays declared with const, their properties can be modified, but the variable itself cannot be reassigned to a different object or array. Hoisted to the top of their block scope but are not initialized. Attempting to access them before their declaration results in a ReferenceError (known as the Temporal Dead Zone or TDZ). Use const for values that should not change.
